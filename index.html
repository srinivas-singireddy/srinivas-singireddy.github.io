Flagship project · Built & operating

MLOps Platform on AWS EKS

A production-grade Kubernetes platform, built from an empty AWS account to a fully reconciled GitOps estate. Two repositories, one contract: Terraform owns infrastructure, ArgoCD owns applications — and they meet at exactly one boundary, the ArgoCD root Application.

mlops-platform-aws-eks

Infrastructure. Every AWS resource in Terraform, modularized by lifecycle: bootstrap → network → cluster → platform.

mlops-platform-deploy

Applications. ArgoCD app-of-apps manifests, continuously reconciled (~3-minute loop, selfHeal enabled). Git is the only source of truth.

Technical highlights · built

Infrastructure as code, by lifecycle

Four Terraform roots split by blast radius and change cadence — the state backend never shares a lifecycle with the cluster it protects.

Deliberate network trade-offs

Two-AZ public/private split with a single NAT Gateway — a documented lab cost decision, with the one-NAT-per-AZ production alternative recorded alongside it.

VPC CNI prefix delegation

Raised pod density from ~17 to ~110 pods per t3.medium — at zero additional cost.

Observability, current-generation

Prometheus, Grafana, Loki, and Alertmanager — with Grafana Alloy adopted as the collector after Promtail reached end of life.

Cost-aware operations

Ephemeral cluster pattern: full destroy/apply daily discipline keeps the lab at ~€20–30/month instead of ~€100/month running 24/7. gp3 as the default StorageClass.

7 architecture decision records and 30+ documented lessons learned — every non-obvious choice has a written why.

Architecture spotlight · built

Secrets without secrets in Git

GitOps has an inherent tension: everything must live in Git, but credentials must not. The platform resolves it with a fully declarative pull path — secrets are authored once in AWS Secrets Manager and materialize inside pods with no plaintext credential ever committed.

The application peak · next tenant workload

dev-deployed · prod-designed

munich-rag-compliance

Compliance-aware Retrieval-Augmented Generation over German regulatory corpora (DSGVO/BDSG), built citation-grounded for the regulated Munich finance and insurance market. It runs today in a dev/local context; its production home is the EKS platform above — that integration is the roadmap, and the reason the platform exists.

Architecture

  • LangGraph agent orchestration — query classifier, conditional routing, parallel multi-article retrieval, citation validator
  • Hybrid retrieval — dense multilingual-e5 + sparse, fused, with cross-encoder reranking
  • EU-sovereign stack — Qdrant vector store, Mistral models
  • Citation-grounded answers — every response traceable to the regulatory article it cites
